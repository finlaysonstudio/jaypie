## Changes

- Renamed `"level"` field to `"status"` in JSON output to prevent Lambda/CloudWatch/Datadog Forwarder from merging it with their own `level` field, which created an array (`["error","ERROR"]`) that Datadog discarded
- Updated DatadogTransport to strip `"status"` instead of `"level"` when forwarding

## Migration

JSON output changes from:
```json
{"level":"debug","message":"hello"}
```
to:
```json
{"message":"hello","status":"debug"}
```

Any log parsing rules that match on the `"level"` field should be updated to match `"status"`.
