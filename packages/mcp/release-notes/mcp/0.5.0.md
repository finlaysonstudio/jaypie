# @jaypie/mcp@0.5.0

## Breaking Changes

All AWS, Datadog, and LLM tools have been consolidated into unified router-style tools. The tool names have changed:

### Previous Tools (26) → New Tools (6)

| Old Tool | New Usage |
|----------|-----------|
| `aws_list_profiles` | `aws("list_profiles")` |
| `aws_lambda_list_functions` | `aws("lambda_list_functions", { ... })` |
| `aws_dynamodb_query` | `aws("dynamodb_query", { ... })` |
| `datadog_logs` | `datadog("logs", { ... })` |
| `datadog_monitors` | `datadog("monitors", { ... })` |
| `llm_debug_call` | `llm("debug_call", { ... })` |
| `llm_list_providers` | `llm("list_providers")` |
| `list_release_notes` | `release_notes("list", { ... })` |
| `read_release_note` | `release_notes("read", { ... })` |

### Tools That Remain Unchanged
- `skill` - Unchanged
- `version` - Unchanged

## New Features

### Progressive Disclosure
All unified tools return help documentation when called with no arguments:
```
aws()           → Returns help markdown with all commands
datadog()       → Returns help markdown with all commands
llm()           → Returns help markdown with all commands
release_notes() → Returns help markdown with all commands
```

### Modular Architecture
Code reorganized into domain-specific directories:
```
src/suites/
├── aws/          # AWS CLI integration
├── datadog/      # Datadog observability
├── llm/          # LLM debugging
└── docs/         # Documentation services
```

Each suite contains:
- `index.ts` - Unified service with command router
- `help.md` - Documentation for the tool
- `<domain>.ts` - Implementation functions

## Migration Guide

1. Replace individual tool calls with unified router pattern
2. Pass the old tool suffix as the `command` parameter
3. Pass parameters in the `input` object

Example migration:
```typescript
// Before
aws_lambda_list_functions({ region: "us-east-1" })

// After
aws("lambda_list_functions", { region: "us-east-1" })
```
