## Changes

- `llm` skill documents the new `effort` option — provider-neutral reasoning
  scale (`lowest|low|medium|high|highest`), the per-provider translation table,
  papered-level `log.debug`, and gating behavior (including OpenAI version-gated
  `minimal`/`xhigh`)
- `llm` skill documents the model constants: modernized default models
  (`PROVIDER.*.DEFAULT`), the `MODEL.*` catalog incl. `MODEL.OPENROUTER`, and
  the deprecation of the size-tier map / `DEFAULT.MODEL` / `ALL`
- `vocabulary` skill redefines `status` as a per-model axis (value vocabulary
  declared by the model) and names the severity/status boundary — log levels
  are severity, never `status`; reserves `level` for severity
- `dynamodb` skill documents the optional `status` vocabulary on `registerModel`
- Release notes for llm 1.3.10 and fabric 0.3.4
