## Changes

- Add `resolveTtl(input)` — resolves a TTL to DynamoDB epoch **seconds**. Accepts a future epoch `number`, a duration string (`"1 day"`, `"4 weeks"`, `"1 month"` = 31 days, `"1 year"` = 366 days), or a standard ISO 8601 date string. A non-future result is still returned but logged at `error`.
- `createEntity({ entity, ttl })` and `updateEntity({ entity, ttl })` accept a per-call `ttl` (`TtlInput | false`). Precedence: `ttl: false` clears any TTL; an explicit `ttl` wins; an existing `ttl` on the entity is preserved; otherwise `createEntity` applies the model's registered default. `updateEntity` never applies the model default, so re-saving does not slide expiry.
- Register a model default with `registerModel({ model, ttl })` (via `@jaypie/fabric`).
- Add `DEFAULT_TTL_ATTRIBUTE` (`"ttl"`) and an explicit `ttl?: number` field on `StorableEntity`.

TTL is best-effort (typically within 48h) and requires TTL enabled on the table — see `JaypieDynamoDb` (`@jaypie/constructs`), which now enables TTL on `ttl` by default.
