## Changes

- New `LLM.COST` catalog in `constants.ts`, a peer to `MODEL`, mapping a model id to `{ input, output, cachedInputRead?, cachedInputWrite?, reasoning? }` in US dollars per **million** tokens. The `LlmModelCost` type is exported from the package index.
- Keys are **string literals**, not `MODEL.*` references. A model retired from the catalog keeps its price, so historic usage records stay replayable. 35 entries cover Anthropic, Fireworks, Google, OpenAI, and xAI, including ids reachable only through the deprecated `PROVIDER.*.MODEL.*` tiers.
- `cachedInputWrite` is a scalar when the rate does not vary by TTL, or keyed `{ "5m", "1h" }` using the same literals as `LlmCache`, so a cost calculation reads the TTL straight off `OperateRequest.cache`. Omitting the field means writes bill at `input` — currently populated for Anthropic only, the one provider publishing a write premium.
- Rates are the standard short-context text tier. Introductory, batch, flex, priority, fast-mode, and data-residency pricing are excluded, as are long-prompt surcharges (Gemini 3.1 Pro above 200K, Grok at 200K, GPT-5.5 above 272K).
- Bedrock and `MODEL.OPENROUTER.*` are deliberately absent: both resell many vendors and price per backend route, so no single rate is correct for a proxy id. Unlisted ids return `undefined`; callers must handle a miss.

## Fixes

- Bedrock: a model that rejects `cachePoint` blocks is now correctly denylisted and retried without them. Bedrock words the rejection "You invoked an **unsupported** model or your request did not allow prompt caching", and the detector only matched `not support` — one word versus two — so the graceful-degradation retry never fired and the call failed outright. Surfaced by the live capability matrix on `deepseek.v3.2` and `moonshotai.kimi-k2.5`.
