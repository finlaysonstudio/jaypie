## Changes

- **`MODEL.NOVA_PRO` and `MODEL.NOVA_LITE`** join the catalog as first-class ids. Amazon's own models are the only ones catalogued for Bedrock; Bedrock's third-party routes stay uncatalogued, reachable by passing the literal id, which `determineModelProvider` resolves to `bedrock`. `PROVIDER.BEDROCK.DEFAULT` and the deprecated size tiers now reference `MODEL.NOVA_*` instead of literals.
- **`MODEL.FIREWORKS.GPT_OSS`** (`accounts/fireworks/models/gpt-oss-120b`) added and priced.
- **`PROVIDER.BEDROCK.MODEL_MATCH_WORDS`** widened with `deepseek.`, `google.gemma`, `moonshotai.`, `openai.gpt-oss`, and `qwen.` so those literal Bedrock ids resolve to the `bedrock` provider.
- **`COST` prices Amazon's Nova models** from the AWS Price List API (`AmazonBedrock/us-east-1`, published 2026-07-20), including a published `cachedInputWrite: 0`. `cachedInputWrite` now documents `0` as a valid scalar. Nova 2 Lite is quoted at the `us.` geo rate the catalog id bills against; the cheaper `global.` profile is not modeled.
- **Gateway routes stay unpriced.** `MODEL.OPENROUTER.*` and any Bedrock id reselling a third-party model cost per route and per region, so no single rate is correct. Amazon's own Nova models are first-class, not proxies, and are priced.

## Removals

- **Fireworks Nemotron retired from `MODEL.*`.** Its structured output is nondeterministic: across six live runs the same request produced clean JSON, prose, and an empty array. Its `COST` entry remains so historic usage stays replayable.

## Testing

- `test/models.ts` derives the entire live capability matrix from `MODEL.*` plus each `PROVIDER.*.DEFAULT` and carries no model ids of its own. `npm run test:matrix:bedrock` now shards by `APP_GROUP=bedrock` rather than an inline id list.
