## Changes

- `persistExchange` now resolves the optional `@jaypie/dynamodb` peer via a native dynamic `import()` instead of `createRequire`. An ESM host initializes the ESM build; the previous `require()` loaded the CJS build, whose module-level client state is separate, so `storeExchange` always saw an uninitialized client and wrote nothing. Both now share one module instance and exchanges persist as intended (#429).
- The dynamic import is bundler-proof (survives emission to both the ESM and CJS `@jaypie/llm` builds without being rewritten to `require()`), so the optional peer is still never statically bundled.
