## Changes

- Throw typed `LlmError` subclasses (`LlmRateLimitError`, `LlmQuotaError`, `LlmUnrecoverableError`, `LlmTransientError`) from the operate/stream retry layer instead of a bare `BadGatewayError`, preserving `provider`, `model`, `retryAfterMs`, and the original error as `cause`.
- Add `ErrorCategory.Quota` and a shared `classifyProviderError` first pass so providers agree on cross-provider conditions: retryable structured-output compile timeouts, exhausted quota, and billing/insufficient-funds. A daily-quota 429 now classifies as `Quota` (terminal), not `RateLimit`.
- Accept `model` as a preference-ordered array: `model: string[]` is sugar for a fallback chain (index 0 primary, later entries fallbacks with provider auto-detected). Explicit `fallback` is appended after the array-derived chain; an empty array throws `ConfigurationError`.
- Export the new error types through `@jaypie/llm`.
