## Changes

- New `parseFrontmatter(content)` and `stringifyFrontmatter(content, data)`
  utilities for reading and writing YAML frontmatter in markdown. Built on
  `js-yaml@4`, they mirror the slice of `gray-matter` Jaypie relied on (default
  `---` delimiters, YAML engine) without its abandoned, `js-yaml@3`-pinned
  dependency.
  - `parseFrontmatter<T>(content)` returns `{ content, data }`; absent
    frontmatter yields `data: {}` and the input body unchanged.
  - `stringifyFrontmatter(content, data)` round-trips back to a frontmatter
    string; an empty `data` object yields the body alone.
- Adds `js-yaml@^4.2.0` as a dependency. `js-yaml@4.2.0` patches the
  quadratic-complexity merge-key DoS (GHSA-h67p-54hq-rp68) present in the
  `js-yaml@3` line gray-matter pulled.

## Migration

No changes required. Consumers replacing direct `gray-matter` use can swap
`matter(content)` → `parseFrontmatter(content)` and
`matter.stringify(content, data)` → `stringifyFrontmatter(content, data)`.
