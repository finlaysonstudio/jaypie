## Changes

- `registerModel({ model, indexes, status })` accepts an optional `status`
  vocabulary — the declared value list for that model's `status` axis. Models
  that omit `status` keep free-string behavior. `registerModel` throws
  `ConfigurationError` when `status` is not a non-empty array of non-empty
  strings.
- New `getModelStatus(model)` — returns the model's declared `status`
  vocabulary, or `undefined` when the model is unregistered or declares none.
- New `isModelStatus(model, value)` — `true` when the model declares no
  vocabulary (any string passes) or `value` is in the declared vocabulary.
- New `assertModelStatus(model, status)` — throws `BadRequestError` when the
  model declares a vocabulary and `status` is not in it; no-op otherwise.

## Vocabulary

- `status` is a per-model axis: the entity's position in its model's lifecycle,
  with the value vocabulary declared by the model rather than a single global
  list. See skill("vocabulary"). Log levels (trace/debug/info/warn/error/fatal)
  are **severity**, never `status`; `level` is reserved for severity.

## Migration

- Purely additive. Existing `registerModel` calls and free-string `status`
  values are unaffected.
